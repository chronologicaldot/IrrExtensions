• position

    Attribute value type: Enumeration (HTMLPosTypes in webtypes.h)
    Default value: "static"

    Not the same as Position, which does nothing.

    This indicates the type of positioning an element will experience, independent of the type of element it is. (Currently, it applies only to GUI elements.) The way it treats elements is analogous to HTML:

    • absolute - positions an object outside of the normal flow of elements but within the boundaries of the parent GUI element (attributes affecting this are "left", "right", "top", "bottom", "height", and "width");
    • fixed - positions an object similar to "absolute", but the parent object is the root GUI element;
    • relative - positions objects within the normal flow of elements but offsets the element to the right by the amount given by the attributes "left" and "top". Furthermore, size judgment is made based on the absolute rectangle of the containing element from the top, not from current placement within the element (attributes affecting this are "left", "right", "top", "bottom", "height", and "width");
    • static - positions objects within the normal flow of elements but offsets the element by the amount given by the attribute "top". Furthermore, size judgment is made based on the absolute rectangle of the containing element from the top, not from current placement within the element (attributes affecting this are "left", "right", "top", "bottom", "height", and "width");

  • left

    Attribute value type: Integer (irr::s32)
    Default value: 0

    Helps define the bounding box of an element by establishing how far left from the parent element's left edge (or preceding element's right edge) this element's left edge is to be placed.

    Current implementation allows you to add "px" to the end of the value because it will be ignored.

  • right

    Attribute value type: Integer (irr::s32)
    Default value: 0

    Helps define the bounding box of an element by establishing how far right from the parent element's right edge this element's right edge is to be placed disregarding how it will be repositioned based on the flow of elements.

    Current implementation allows you to add "px" to the end of the value because it will be ignored.

  • top

    Attribute value type: Integer (irr::s32)
    Default value: 0

    Helps define the bounding box of an element by establishing how far below the parent element's top edge (or preceding element's bottom edge) this element's top edge is to be placed.

    Current implementation allows you to add "px" to the end of the value because it will be ignored.

  • bottom

    Attribute value type: Integer (irr::s32)
    Default value: 0

    Helps define the bounding box of an element by establishing how far above the parent element's bottom edge this element's bottom edge is to be placed disregarding how it will be repositioned based on the flow of elements.

    Current implementation allows you to add "px" to the end of the value because it will be ignored.

  • height

    Attribute value type: Integer (irr::s32)
    Default value: 0

    Helps define the bounding box of an element by establishing its height. If the top is given and not the bottom, the height will determine the bottom edge of the bounding box, and vice versa (bottom given instead of top). This attribute will be ignored if both the top and bottom attributes are found.

    Current implementation allows you to add "px" to the end of the value because it will be ignored.

  • width

    Attribute value type: Integer (irr::s32)
    Default value: 0

    Helps define the bounding box of an element by establishing its width. If the left is given and not the right, the width will determine the right edge of the bounding box, and vice versa (right given instead of left). This attribute will be ignored if both the left and right attributes are found.

    Current implementation allows you to add "px" to the end of the value because it will be ignored.
